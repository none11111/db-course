# 论坛数据分析平台演示脚本

## 演示目标

展示论坛数据分析平台的核心功能和技术亮点，包括数据可视化仪表盘、API接口调用、性能优化成果等。

## 演示环境准备

### 1. 检查数据库连接
确保MySQL服务正在运行，且数据库已正确配置。

### 2. 初始化测试数据
运行应用将自动创建测试数据：
```bash
python run.py
```

### 3. 确认服务启动
访问 http://10.157.174.70:5000 确认应用正常运行。

## 演示流程

### 阶段一：系统概览介绍（2分钟）

**1. 首页展示**
- 访问 http://localhost:5000/
- 介绍系统的设计理念：后端为王，数据为本
- 简要说明平台的核心功能模块

**2. 技术架构简介**
- 后端：Flask + SQLAlchemy + MySQL
- 前端：HTML5 + Bootstrap 5 + ECharts
- 强调数据库设计的重要性

### 阶段二：数据仪表盘展示（5分钟）

**1. 进入仪表盘页面**
- 导航至 http://localhost:5000/dashboard

**2. 概览统计卡片**
- 展示总用户数、总帖子数、总评论数等核心指标
- 说明数据来源和更新机制

**3. 月度发帖趋势图表**
- 解释图表的含义：展示论坛活跃度随时间的变化
- 演示如何识别社区活跃高峰期
- 简要说明背后的SQL查询逻辑

**4. 帖子分类分布饼图**
- 展示不同类型内容的占比
- 解释分类统计的业务价值
- 提及使用索引优化提高查询效率

**5. 核心用户排行柱状图**
- 展示对社区贡献最大的用户
- 介绍用户声望计算机制
- 提及使用存储过程批量计算用户声望

**6. 用户活跃度分布图**
- 解释如何识别不同类型的用户群体
- 说明这对社区运营策略的指导意义

### 阶段三：API接口演示（3分钟）

**1. 展示API文档**
- 简要介绍API的设计原则：RESTful风格

**2. 演示API调用**
- 使用浏览器或Postman访问以下API：

  ```
  # 获取论坛概览数据
  http://localhost:5000/api/overview
  
  # 获取月度发帖趋势
  http://localhost:5000/api/monthly-posts
  
  # 获取核心用户排行
  http://localhost:5000/api/top-users
  
  # 获取帖子分类分布
  http://localhost:5000/api/category-distribution
  ```

**3. 说明API实现细节**
- API如何调用数据库视图和存储过程
- 数据序列化和响应格式

### 阶段四：性能优化成果展示（3分钟）

**1. 运行性能测试脚本**
```bash
python app/utils/performance_test.py
```

**2. 解读性能测试报告**
- 展示优化前后的查询执行时间对比
- 强调索引优化的效果
- 展示存储过程与普通查询的性能差异

**3. 展示索引使用情况**
- 演示使用EXPLAIN分析查询执行计划
```sql
EXPLAIN SELECT * FROM posts WHERE category = '技术讨论' ORDER BY created_at DESC;
```

### 阶段五：数据验证与完整性检查（2分钟）

**1. 展示数据验证脚本**
- 简要说明 `data_validation.sql` 的内容和用途
- 演示部分验证查询的执行

**2. 展示存储过程功能**
- 演示更新用户声望的存储过程
```sql
CALL UpdateUserReputation(1);
```

- 演示获取热门帖子的存储过程
```sql
CALL GetHotPosts(10);
```
